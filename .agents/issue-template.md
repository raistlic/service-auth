# Issue Template

## Context

Describe the surrounding product, workflow, customer need, or technical background that makes this issue necessary.

## Problem

Describe the current gap, bug, risk, or missing capability in concrete terms.

## Requirements

- List the required behavior or outcome.
- Include important constraints or non-goals.
- Keep each requirement specific and testable.

## Definition of Done

- State the observable completion criteria.
- Focus on outcomes that make the issue ready to close.
- Include documentation, UX, or operational expectations when relevant.

## Testing Guide

- Describe how the implementation should be verified.
- Include commands, environments, scenarios, or manual checks when known.
- Note any important edge cases.

## Additional Notes

Capture optional supporting details such as links, screenshots, related PRs, related issues, or open questions.
